模型的评估和优化

当你根据你的数据训练出来了一个模型，你希望知道你的模型表现如何。在前面的文章中我们有提到过度拟合(overfit) 和 拟合不足(underfit)的情况，这都表示模型不太能很好的进行预测。在你发现模型的问题之后，需要进行一系列的措施来优化模型，这需要根据模型的具体问题来决定。评估和优化都是训练中很重要的部分，能够帮助你更快的训练出更好的模型。

一、评估模型

1、高方差，高偏差

假设现在你手上有20份数据，你要训练的是一个只有一个特征值的线性回归模型。那么根据特征工程，你的模型可能是以下几种

f(x)=wx+bf(x)=w1x+w2x2+bf(x)=w1x+w2x2+w3x3+b...f(x)=w1x+w2x2+w3x3+...+w7x7+bf(x) = wx+b\\ f(x)=w_1x+w_2x^2+b\\ f(x)=w_1x+w_2x^2+w_3x^3+b\\ ...\\ f(x)=w_1x+w_2x^2+w_3x^3+...+w_7x^7+b f(x)=wx+bf(x)=w1​x+w2​x2+bf(x)=w1​x+w2​x2+w3​x3+b...f(x)=w1​x+w2​x2+w3​x3+...+w7​x7+b

如果你选择了f(x) = wx + b

那么他的拟合效果可能不太好，因为他不仅在现有的数据上表现一般，在新数据的预测上表现也会一般。这时候是拟合不足，我们也称模型具有高方差(high bias)

如果你选择了一个非常高阶的模型，能够适应所有的数据集

这时候拟合效果也不太好，因为虽然它能够适应所有现有的数据，但是在新数据上的表现并不好。这时候是过度拟合， 也称模型具有高偏差(high variance)

  • 高方差(high bias) - 拟合不足 (underfit)
  • 高偏差(high variance) - 过度拟合 (overfit)

无论是具有高方差还是高偏差的模型，都不是一个效果很好的模型，都需要进行优化。

2、训练集、测试集

那么如何知道我们训练的模型是否具有高方差和高偏差呢？评估需要知道模型在新数据上的表现，但我们手上并没有更多的数据。

那我们就可以把我们手上的数据分为两份，一份训练集，一份测试集。

训练集大概在70%， 测试集大概在30%。

我们在使用训练集训练完模型之后， 可以用训练集和测试机分别计算模型的成本。

Jtrain=12mtrain∑i=1mtrain(f(xi)−ytraini)2Jtest=12mtest∑i=1mtest(f(xi)−ytesti)2J_{train} = \frac{1}{2m_{train}} \sum_{i=1}^{m_{train}}(f(x^i) -y_{train}^i)^2 \\ J_{test} = \frac{1}{2m_{test}} \sum_{i=1}^{m_{test}}(f(x^i) -y_{test}^i)^2 \\ Jtrain​=2mtrain​1​i=1∑mtrain​​(f(xi)−ytraini​)2Jtest​=2mtest​1​i=1∑mtest​​(f(xi)−ytesti​)2

接下来我们来看一下， 当模型具有高方差，也就是拟合不足的时候:

这时候因为模型在原本的训练集上表现也不佳， 在测试集上表现也不佳，所以这时候

Jtrain  is  highJtest  is  highJ_{train} \ \ is \ \ high\\ J_{test} \ \ is \ \ high Jtrain​  is  highJtest​  is  high

J_train和J_test的值都很大。

而当模型具有高偏差，也就是过度拟合的时候

因为模型几乎完美适应了训练集的数据，而在新数据上表现不佳，所以有

Jtrain≈0Jtest>>Jtrain { } J_{train}\approx0\\ J_{test} >>J{train} Jtrain​≈0Jtest​>>Jtrain

J_train几乎等于0， 而J_test要比J_train要大得多。

如果模型是一个正好的模型，那么应该有

Jtrain  is  lowJtrain≈JtestJ_{train} \ \ is \ \ low\\ J_{train} \approx J_{test} Jtrain​  is  lowJtrain​≈Jtest​

J_train和J_test都很小。

所以我们可以得到

  • J_train is high, J_test is high -> 高方差
  • J_train ≈ 0, J_test is high -> 高偏差
  • J_train is log, J_test is low -> 刚好

通过J_train和J_test我们就可以评估一个模型的好坏。

当你不知道应该用几阶的多项式作为模型时，就可以计算出每个模型的J_train和J_test， 选出其中最佳的模型。

f(x)=wx+bf(x)=w1x+w2x2+bf(x)=w1x+w2x2+w3x3+b...f(x)=w1x+w2x2+w3x3+...+w7x7+bf(x) = wx+b\\f(x)=w_1x+w_2x^2+b\\f(x)=w_1x+w_2x^2+w_3x^3+b\\...\\f(x)=w_1x+w_2x^2+w_3x^3+...+w_7x^7+b f(x)=wx+bf(x)=w1​x+w2​x2+bf(x)=w1​x+w2​x2+w3​x3+b...f(x)=w1​x+w2​x2+w3​x3+...+w7​x7+b

比如你算出来f(x) = w1x + w2x² + w3x³ + b模型的J_train和J_test都比较低，那么就可以选择这个模型。

3、交叉验证集

在上一节我们将数据集一分为二，分成了训练集和测试集，并且分别计算模型在两个数据集上的成本来判断模型的好坏。

然而这个方法有一定的局限，就是有一定的运气成分在里面。可能模型刚好在这部分测试集上表现也良好， 但是在其他更多的数据上表现并不好。 也就是说，这样方法选择出来的是我们认为好的模型。

为了解决，我们可以将数据集再划分，引入交叉验证集(cross validation)， 一分为3:

  • 训练集 占60%
  • 交叉验证集 占20%
  • 测试集 占20%

然后，我们用训练集训练模型，然后分别计算该模型在训练集和交叉验证集的成本，选出此时最优的模型。最后再计算这个模型在测试集上的表现。

可以理解为交叉验证只是一次内部测试，这一次测试可能只能发现一些小问题。而最后的测试集是一次外部测试，可能会发现更多问题。

用这样一分为三的的方式，使我们评估的模型更具有一般性。

你可能在比较的时候，发现

f(x)=w1x+w2x2+w3x3+bf(x)=w1x+w2x2+w3x3+w4x4+w5x5+bf(x)=w_1x+w_2x^2+w_3x^3+b \\ f(x)=w_1x+w_2x^2+w_3x^3+w_4x^4+w_5x^5+b f(x)=w1​x+w2​x2+w3​x3+bf(x)=w1​x+w2​x2+w3​x3+w4​x4+w5​x5+b

这两个模型，他们的J_train和J_cv(交叉验证集的成本)都比较低， 其中三阶的更低一点。

于是你用三阶模型在测试集上验证， 结果发现得到的J_test非常高。这时候再回过头用五阶模型在测试集上验证，发现J_test要小很多。那么说明五阶模型应该比三阶模型更优。

4、正则化与偏差/方差

倘若你现在模型已经定了， 有三个特征值的模型

f(x⃗)=w1x1+w2x2+w3x3+bf(\vec{x}) = w_1x_1 + w_2x_2+w_3x_3+b f(x)=w1​x1​+w2​x2​+w3​x3​+b

那么，虽然无法从特征工程上选择模型，但我们可以通过正则化来优化模型，从而降低某个特征值的权重。

我们知道，正则化是对成本函数进行了修改

J(w,b)=12m∑i=1m(f(x⃗)−y)2+λ2m∑j=1nwjJ(w, b) = \frac{1}{2m}\sum_{i=1}^m(f(\vec{x}) - y)^2 + \frac{\lambda}{2m}\sum_{j=1}^nw_j J(w,b)=2m1​i=1∑m​(f(x)−y)2+2mλ​j=1∑n​wj​

我们来讨论一下，在这个过度拟合的模型上进行正则化

  • 当λ非常小， ≈0的时候，此时后面加上的式子就≈0， 相当于没有进行正则化。这时候的模型是过度拟合的， 也就是具有高偏差。

  • 而当λ非常大时，正则化会把所有的w都减少到一个很小的值， 导致最后的模型是f(x) = b. 此时模型就成了拟合不足，也即是具有高方差。

    所以λ需要在一个合适的值上，模型才会表现好。

根据上面的讨论，我们可以得出结论

  • 当λ=0时，模型具有高偏差， J_train小，J_cv也很大
  • 当λ非常大时，模型具有高方差， J_train很大， J_cv也很大。

所以，我们在进行正则化的时候， 也可以通过J_train和J_cv来寻找一个合适的λ

5、确定基准水平

通过之前的讨论，我们知道可以通过 J_train和J_cv来评估一个模型的优劣。但是我们并不知道J_train多高算高？J_cv多高算高？所以我们需要一个基准水平。

而这个基准水平通常就是人类能达到的水平。

比如在语音识别的场景下，人类听一段语音识别其中的文字错误率大概在10%左右。

人类水平 J_train J_cv 结论
10% 10.8% 15.5% 高偏差
10% 15.5% 16.0% 高方差

通过和人类的能力相比，我们就可以知道J_train和J_cv的合理范围。超出这个范围过多时， 就需要检查模型是否具有高偏差或高方差了。

6、学习曲线

如果你的模型出现了高偏差，在之前的笔记中我们有提到可以通过增加训练数据集的方式来优化模型。 我们来看看在这样的情况下增加训练数据集与J_train J_cv的关系。

当增加数据集时，模型会努力去拟合所有训练数据， 这时候J_train可能会增大，但同时J_cv会慢慢变小。

因为是高偏差，初始的J_train非常小，在数据集上的表现比基准线还好，但是J_cv的成本很高。

随着数据集的增加，J_train和J_cv都不断向基准线靠拢。这个过程，在有更多训练数据情况下， 模型会调整各个特征值的系数，更具有一般性。

但是，如果你的模型具有高方差，增加训练数据的方式就显得不那么奏效了。

比如你选择了w1+b作为模型，但实际并不是一个正比例的关系。这时候你增加数据集，因为出现更多数据，模型无法拟合，反而会不断提高模型在训练集上的成本。

同时，模型为了努力的拟合数据，增加数据会使得模型在交叉验证集上的表现变好。

这时候画出他们的关系图， 大概可能是这样

你会看到，通过增加数据， 你没有办法找到一个J_train和J_cv都很小的点，他们最终都会高于基准线。

这是因为，你的模型表达式不正确，增加再多的数据也是徒劳。

所以在拟合不足的情况下增加数据无法优化模型， 更多的可能要通过特征工程来调整模型。

7、偏差/方差与神经网络

从前面的讨论中我们不难发现，当你的模型多项式比较简单的时候，往往模型具有高方差；而多项式过于复杂的时候，往往模型具有高偏差。

f(x)=wx+b  high biasf(x)=w1x+w2x2+w3x3+b  just goodf(x)=w1x+w2x2+w3x3+w4x4+w5x5+b   high variancef(x) = wx+b \ \ high\ bias\\ f(x) = w_1x + w_2x^2 +w_3x^3 + b \ \ just \ good\\ f(x) = w_1x + w_2x^2 +w_3x^3 + w_4x^4 +w_5x^5 + b \ \ \ high\ variance f(x)=wx+b  high biasf(x)=w1​x+w2​x2+w3​x3+b  just goodf(x)=w1​x+w2​x2+w3​x3+w4​x4+w5​x5+b   high variance

所以，给模型选择一个合适的多项式是非常重要的。

这也可以推理到神经网络里:

  • 当你的神经网络过于简单的时候，无论在训练集还是在交叉验证集上神经网络的表现都不太好。
  • 当你的神经网络过于复杂的时候，他几乎能完美拟合训练集，但是在交叉验证及上表现不佳。

所以在构建你的神经网络的时候， 也要避免高方差和高偏差的出现。

二、优化模型

知道了怎么评估模型，下一步就是如何对模型进行优化了。

1、优化高方差模型

高方差模型是拟合不足，所以我们可以采用以下几种方式来优化：

  • 使用特征工程，增加特征值的数量。

    高方差的模型往往是因为训练的特征值不足以进行拟合，所以我们可以适当增加几个特征值，如在房价预测中除了房屋的面积以外，房屋的地段、楼层可能都会有影响。

  • 多项式回归

    如果没有更多的特征值可以选，我们也可以在模型的多项式里通过组合、升次来修复高方差。比如x², x³…

  • 适当减小λ

    在正则化的时候，适当减小λ能够提高回归参数， 也就是能提高特征的比重。这时候也能帮助修复高方差。

2、优化高偏差模型

高偏差模型为过度拟合，我们可以通过下面的方式进行优化:

  • 获取更多的训练数据

    在前面的学习曲线中我们已经提到过，获取更多数据能够帮助优化高偏差。

  • 尝试更少的特征值集合

    可以尝试去掉多项式里的一些项，比如x^5, x_1x_2… 过度拟合往往也是多项式过于复杂导致的。

  • 适当增大λ

    如果无法去掉特征值，那么可以尝试在正则化的时候增大λ。

3、误差分析

假设我们现在训练一个识别一封邮件是否是垃圾邮件的模型。

一封垃圾邮件可能有以下特征:

  • 文中是否有”购买“，”优惠“，”信用卡“ 等词语
  • 文中是否有错别字
  • 文中是否有带跳转链接的图片
  • …

在经过训练和上面的优化之后，发现模型的效果依然不好， 接下来应该怎么办呢？

我们可以从交叉验证数据集里取出那些验证错误的例子出来，并人工对他们进行分析。假设我们取了其中100个数据分析，分析结果如下:

  • 有72封文中有”购买“，”优惠“，”信用卡“ 等词语
  • 有20封文中是否有错别字
  • 有3封文中是否有带跳转链接的图片
  • …

我们发现，错误的数据中对于文中有”购买“，”优惠“，”信用卡“ 等词语的邮件识别不太好， 这就是我们模型的问题所在。

而这个时候，我们可以做的就是增加这一类的训练数据集，让模型能更好的拟合这类情况。这比我们盲目地增加数据集要高效得多。

这个过程我们就称作误差分析。

4、数据增强

在模型遇到高偏差的时候，我们会通过增加训练数据的方式来优化模型。但有时候我们已经没有办法获取更多的数据了，这时候应该怎么办呢？

例如， 我们在实现语音识别的模型时，我们有着一段原始音频。这时候，我们可以在原始音频里加上火车的轰鸣声、人群的嘈杂声、汽车的引擎声… 通过这样的方式，我们就能扩大我们的数据集。 并且，如果你在误差分析中发现，你的模型在人群嘈杂的情况下识别效果很差， 那你就可以通过合成多段有不同人群嘈杂声的训练音频。 这就叫做数据增强。

又或者，你在实现一个识别手写26个字母的模型。对于”A“， 你可以手动画出一个放大版的、缩小版的、稍微有点倾斜的A的图片，又或者你可以对原图片添加一些噪声。通过这样的方式，你就能得到更多的训练数据。

你还可以把A放在一个网格中， 通过变化网格来得到不同形状的A

甚至你可以在PPT里使用不同的艺术字来画A， 然后添加到训练数据里。

数据增强配合上误差分析，能够让你在只添加了少部分数据的情况下大大优化模型的效果。

在机器学习刚兴起的时候，研究人员们需要优化算法(Code)和获取数据(Data)，侧重在代码实现上； 而到现在很多算法已经很优秀了， 使得研究人员可以把重心放到获取数据(Data)上去。

所以，数据增强也能够对我们的模型进行优化。
